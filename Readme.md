# Investment Database

## О проекте
Этот проект — мой pet-project и одновременно личный инструмент для учета инвестиций.

## Цель
Перейти от Excel к структурированной системе на PostgreSQL с возможностью дальнейшей аналитики и визуализации.

## Технологии
- PostgreSQL
- Docker
- SQL
- Python

## Текущая структура
Проект включает таблицы:
- accounts — инвестиционные счета
- assets — финансовые инструменты
- trades — сделки покупки и продажи
- cash_transactions — денежные операции по счету
- prices — исторические и актуальные рыночные цены

## Загрузка цен
В проекте есть скрипт [load_prices.py](load_prices.py), который загружает рыночные цены из MOEX ISS в таблицу `prices`.

### Что делает скрипт:
- подключается к PostgreSQL через переменную окружения `DATABASE_URL`
- выбирает из таблицы `assets` инструменты с заполненными `moex_secid` и `board_id`
- запрашивает дневные свечи MOEX по акциям, облигациям и фондам
- для облигаций нормализует цену из процентов от номинала в рубли через `FACEVALUE`
- сохраняет нормализованную цену закрытия в таблицу `prices`
- при повторном запуске обновляет данные через `upsert`, а не создает дубликаты

## Режимы запуска:
- `python3 load_prices.py history 365` — загрузить историю за указанное количество дней
- `python3 load_prices.py latest` — загрузить только последнюю доступную цену по каждому активу

## Автоматизация:
- обновление последних цен настроено через `cron`
- типичный запуск по расписанию использует режим `latest`

## Аналитический слой
В проекте реализованы materialized views и views для расчета текущего состояния портфеля:
- `v_positions_current` — текущие открытые позиции
- `v_portfolio_current` — текущая рыночная витрина портфеля
- `v_portfolio_weights` — веса бумаг в портфеле
- `v_trade_wac` — технический слой расчета себестоимости по WAC
- `v_realized_pnl_trades` — реализованный результат по сделкам продажи
- `v_realized_pnl_by_asset` — агрегированный realized P&L по бумаге
- `v_realized_pnl_by_account` — агрегированный realized P&L по счету

## WAC-логика
1. В расчет входят только исполненные сделки из trades.
2. Для каждой бумаги сделки обрабатываются в хронологическом порядке.
3. После покупки:
   new_qty = old_qty + buy_qty
   new_cost = old_cost + buy_qty * buy_price
   new_avg_cost = new_cost / new_qty
4. После продажи:
   new_qty = old_qty - sell_qty
   avg_cost не меняется
   realized_pnl = (sell_price - avg_cost) * sell_qty
   remaining_cost = avg_cost * new_qty
5. unrealized_pnl считается только на текущий остаток позиции:
   unrealized_pnl = (market_price - avg_cost) * current_qty
6. Если current_qty = 0, позиция считается закрытой.
